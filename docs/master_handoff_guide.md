# Master Handoff Guide

Эта памятка нужна для нового отдельного чата, в котором Codex помогает мастеру **генерировать ходы**, **добавлять миссии** и **обрабатывать результаты** без потери контекста.

## Зачем это нужно

В новом чате Codex не помнит автоматически:

- текущий этап кампании;
- активный открытый ход;
- последние экспортированные результаты;
- локальные договоренности по стилю миссий;
- исключения по наградам, экономике и UI.

Поэтому новый чат нужно запускать не “с нуля”, а через короткий и понятный handoff.

---

## Что дать Codex в начале нового чата

В первом сообщении нового рабочего чата всегда дай:

1. **какая задача сейчас нужна**:
   - сгенерировать новый ход;
   - добавить миссии в уже открытый ход;
   - обработать `export.json`;
   - пересобрать `result.json`;
   - проверить YAML/JSON перед загрузкой в бота;
   - подготовить безопасный деплой правки.

2. **какой сейчас актуальный контекст игры**:
   - номер текущего хода;
   - открыт ли ход сейчас;
   - если открыт, нужны ли новые миссии в него;
   - если ход закрыт, какой `export.json` является актуальным.

3. **какие файлы считать источником истины**:
   - актуальный `turn.yaml` или `append_open_turn.yaml`;
   - актуальный `export.json`;
   - актуальный `result.json`, если он уже собирался;
   - при необходимости `chronicle.md`.

4. **какие ограничения важны именно сейчас**:
   - нужный разброс сложности;
   - нужны ли beginner-миссии;
   - есть ли boss/phased boss;
   - есть ли мастерские исключения по наградам;
   - нужно ли соблюдать серверную совместимость, если Railway еще не догнал новые правила.

## Минимальный стартовый промпт для нового чата

Если нужен **новый рабочий чат по ходу**, можно начинать вот так:

```text
Мы продолжаем вести Telegram RPG Aventura в этом чате.

Считай источниками истины:
- README.md
- prompts/codex_operator_guide.md
- prompts/gm_instructions.md
- prompts/mission_generation_instructions.md
- rules.md

Текущая задача: <сформулируй задачу>.

Текущий контекст:
- текущий ход: <номер/название>
- статус: <открыт / закрыт / нужен результат / нужен append>
- основной файл: <путь к yaml/json>
- дополнительные файлы: <пути, если есть>

Работай осторожно:
- не ломай текущий открытый ход;
- не меняй лишние файлы;
- если нужен деплой, сначала готовь минимальный безопасный пакет.
```

## Если нужно сгенерировать новый ход

В новый чат дай:

1. тему хода;
2. нужное число миссий;
3. пожелания по разбросу сложности;
4. если есть — прошлый `result.json` или `chronicle.md`;
5. если есть открытый вопрос по ростеру — опиши его отдельно.

### Шаблон

```text
Нужно сгенерировать новый ход.

Тема:
<тема хода>

Требования:
- <число> миссии
- одна миссия для слабых/новичков
- одна средняя
- одна верхняя
- если уместно, сделать одну миссию нестандартной

Учитывай контекст:
- последний результат: <путь>
- хроника: <путь, если нужен>

Сохрани результат как turn.yaml в workbench и дай мне путь к файлу.
```

---

## Если нужно добавить миссии в уже открытый ход

Это отдельный режим.

Нужно сразу сказать:

- что ход **уже открыт**;
- что его **нельзя сбрасывать**;
- сколько миссий надо добавить;
- какой они должны быть сложности и темы.

### Шаблон

```text
Нужно добавить миссии в уже открытый ход.

Ход уже открыт. Сбрасывать его нельзя.

Нужно:
- добавить <число> миссии
- тема: <тема>
- сложность: <описание диапазона>
- дополнительные условия: <если есть>

Собери append_open_turn.yaml и дай мне путь к файлу.
```

---

## Если нужно обработать экспорт хода

Нужно дать:

1. путь к `export.json`;
2. если есть — путь к уже собранному, но не принятому `result.json`;
3. если есть особые мастерские решения — описать их прямо текстом;
4. если Railway отстает по логике сервера — обязательно предупредить.

### Шаблон

```text
Нужно обработать экспорт хода и собрать result.json.

Файл экспорта:
- <путь к export.json>

Дополнительно:
- если нужно, учитывай предыдущую неудачную версию result.json: <путь>
- мастерские исключения: <перечень>
- важно сохранить серверную совместимость с текущим Railway, если есть старые валидаторы

Собери валидный result.json, проверь его локально и дай мне путь к итоговому файлу.
```

## Если Railway не принимает файл

В новом чате очень важно сразу сказать, **что именно вернул бот**.

Не писать просто:

```text
не работает
```

А писать так:

```text
Бот вернул ошибку:
<точный текст ошибки>

Файл:
<путь>
```

Это резко ускоряет поиск причины.

---

## Какие файлы лучше прикладывать в новый чат

По убыванию пользы:

1. актуальный `export.json`
2. актуальный `turn.yaml` или `append_open_turn.yaml`
3. последний `result.json`, если он уже собирался
4. `chronicle.md`, если речь о новом ходе
5. скрин или текст ошибки Railway / Telegram-бота

---

## Когда просить именно код, а не просто файл хода

В новом чате отдельно отмечай, что нужен **кодовый фикс**, если:

- бот не принимает JSON/YAML из-за серверной логики;
- нужно менять лимиты участников;
- нужно менять экономику;
- нужно добавлять новую команду;
- нужно править UI Telegram-кнопок;
- нужно безопасно выкатить патч в Railway.

### Шаблон

```text
Нужен кодовый фикс, который пойдет в Railway.

Задача:
<что сломано или что нужно изменить>

Важно:
- сделай минимальный безопасный патч
- перед push прогоняй scripts/predeploy_check.sh
- отдельно скажи, какие файлы коммитить
```

---

## Что Codex должен сделать в хорошем ответе

В новом рабочем чате хороший результат — это когда Codex:

1. понял, что именно сейчас нужно: `turn.yaml`, `append_open_turn.yaml`, `result.json` или кодовый фикс;
2. использовал **актуальные файлы**, а не старые воспоминания;
3. дал **полный путь** к результату;
4. если нужен деплой — назвал **минимальный безопасный набор файлов**;
5. если есть риск, обозначил его до выката.

## Короткая версия handoff

Если нужно совсем коротко, можно начинать новый чат так:

```text
Продолжаем Aventura.

Источники истины:
- README.md
- prompts/codex_operator_guide.md
- prompts/gm_instructions.md
- prompts/mission_generation_instructions.md
- rules.md

Текущая задача:
<что нужно>

Актуальные файлы:
- <путь 1>
- <путь 2>
- <путь 3>

Важно:
- <открытый ход / не сбрасывать / нужен append / нужен result / нужен безопасный деплой>
```

Этого достаточно для большинства рабочих запусков.
