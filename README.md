# Aventura Telegram RPG Bot

Локальный MVP для асинхронной текстовой RPG в Telegram.

Источник истины один: локальная SQLite-база `data/aventura.sqlite`.
Игроки не ведут отдельные листы вручную, а смотрят актуальные данные через команды бота.

Архитектура заложена под несколько режимов игры:

- основной PvE-бот с миссиями;
- будущий PvP-бот для дуэлей и турниров;
- будущий режим фермы гильдии для развития Каррок Манора и инфраструктуры Авентуры.

Все режимы используют одну таблицу героев `characters`, но хранят собственное состояние в отдельных таблицах.

Игровой цикл:

1. Главный мастер готовит `turn_seed.yaml` или просто описывает тему хода.
2. Бот сохраняет seed в `data/seeds/` и готовит workbench в `data/workbench/`.
3. Codex генерирует готовый `turn.yaml`, сверяясь с энциклопедией, базой персонажей и указаниями мастера.
4. Codex добавляет в `turn.yaml` `turn.art.prompt` для общей иллюстрации хода.
5. Если мастер хочет картинку на старте, он генерирует арт и отправляет его боту с подписью `/turn_art`.
6. Админ загружает боту `turn.yaml`.
7. Бот открывает ход, создает миссии и, если арт был загружен, публикует картинку перед миссиями.
8. Бот рассылает список миссий всем игрокам, которые уже нажали `/start`.
9. Игроки выбирают миссию и отправляют действие.
10. В `deadline` бот сам закрывает ход, сохраняет экспорт в `data/exports/`, присылает JSON админу и готовит `data/workbench/turn_<id>/`.
11. Главный мастер при необходимости кладет в workbench `resolution_notes.yaml`.
12. Codex обрабатывает экспорт и `resolution_notes`, затем готовит `result.json`.
13. Codex или мастер кладет `result.json` в `data/imports/pending/` либо админ загружает файл боту.
14. Бот делает бэкап базы, импортирует результат, обновляет городскую хронику и публикует итоги игрокам.

## Мастерские указания

Генерация миссий поддерживает несколько уровней детализации:

- если мастер ничего не уточнил, Codex генерирует миссии сам;
- если мастер указал тему, Codex ориентируется на тему;
- если мастер дал наброски миссий, Codex сохраняет их и дополняет недостающее;
- если мастер указал конкретную сложность или награды, Codex берет их в работу и дополняет только недостающие поля.

При обработке хода мастер может приложить `resolution_notes.yaml`.
Эти комментарии имеют высший приоритет: можно указать, кому дать или не дать награду, какие события обязательны, какие последствия нужно сохранить.

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

В `.env` нужно заполнить:

```text
TELEGRAM_BOT_TOKEN=токен от BotFather
ADMIN_TELEGRAM_IDS=твой_numeric_telegram_id
```

## Запуск

```bash
python -m aventura_bot.bot
```

Бот работает через long polling, поэтому публичный сервер и webhook не нужны.

## Обязательная Проверка Перед Выкатом

Перед каждым `git push`, который должен попасть в Railway, обязательно запускай:

```bash
./scripts/predeploy_check.sh
```

Этот шаг обязателен для любых правок бота и связанных сервисов. Он:

- показывает список измененных файлов;
- прогоняет `py_compile` по главным Python-модулям;
- проверяет импорт `aventura_bot.bot`, чтобы не пропустить сломанный импорт или незакоммиченный связанный файл.

Минимальная безопасная процедура выката теперь такая:

1. Внести правки локально.
2. Запустить `./scripts/predeploy_check.sh`.
3. Если проверка зеленая, сделать `git add`, `git commit`, `git push`.
4. После деплоя в Railway проверить в Telegram:
   - `/start`
   - `Профиль`
   - `Герой`
   - `Миссии`
   - `Мой ход`
   - `Лавка`

Если `predeploy_check.sh` падает, выкатывать правки нельзя, пока причина не исправлена.

## Первый ручной тест

1. Напиши боту `/start`.
2. Создай персонажа:

```text
/create_character Рион | мужской | человек | Молодой авантюрист в сером плаще, привыкший замечать мелкие трещины в людях и вещах. | сила=6 ловкость=6 интеллект=5 харизма=4 восприятие=6 удача=3 | Искра стекла | кинжал, плащ, мел
```
3. При желании сгенерируй картинку по `turn.art.prompt` из `examples/turn.sample.yaml` и отправь ее боту с подписью `/turn_art`.
4. Загрузи файл `examples/turn.sample.yaml` в чат с ботом.
5. Выбери миссию: `/join 1`.
6. Отправь действие, можно с абзацами:

```text
/action Рион осматривает мастерскую и ищет следы зачарованного стекла.

Он не трогает зеркала руками, а отмечает мелом те, где отражение запаздывает.
```

7. Вызови `/export_turn`.
8. Забери JSON из `data/exports/` и обработай его как мастер.
9. Загрузи готовый `result.json`.
10. Вызови `/publish_results 1`.

## Команды игрока

```text
/start
/create_character Имя | Пол | Раса | описание | характеристики | заклинание | предмет1, предмет2, предмет3
/profile
/sheet
/list
/inventory
/spells
/allies
/log
/export_sheet
/missions
/join <id миссии>
/action <текст действия>
/my_action
/shop
/buy <ID товара>
/sell_item <ID предмета>
/sell_pet <имя питомца>
/sell_mount <имя маунта>
/trade @username
/offer_item <ID предмета>
/offer_pet <имя питомца>
/offer_mount <имя маунта>
/remove_item <ID предмета>
/remove_pet <имя питомца>
/remove_mount <имя маунта>
/offer_gold <дублоны>
/trade_status
/accept_trade
/cancel_trade
```

Ответ игрока на миссию лучше писать в 3-4 абзаца по 2-3 предложения.
Действие должно быть логически связано с миссией и пытаться решить ее конфликт.

Обмен между игроками идет только через бота. Передавать можно предметы, питомцев, маунтов и дублоны.
Спутников, заклинания, уровни и характеристики передавать нельзя.
Предметные ID видны в `/inventory`, имена питомцев и маунтов - в `/allies` и `/sheet`; сделка завершается только после `/accept_trade` от обоих участников.
Лавка покупает у игроков предметы, питомцев и маунтов: `/shop`, `/buy <ID товара>`, `/sell_item <ID предмета>`, `/sell_pet <имя>`, `/sell_mount <имя>`.
Каждый новый ход обновляет 25% системного ассортимента лавки; товары игроков, выставленные через продажу, не ротируются автоматически.

На старте у героя нет питомца, спутника и маунта. Стартовое заклинание и три стартовых предмета имеют 1 уровень.
Описание героя - один краткий художественный абзац; оно помогает писать персональные результаты, но не дает механического бонуса.

Прокачка:

- участие без личной победы: +1 к уровню героя;
- личная победа в проверке на completed-миссии: +2 к уровню героя;
- бонусы не суммируются, герой получает либо +1, либо +2;
- completed-миссия дает награду только тем участникам, кто лично прошел проверку;
- обычный пул награды: предмет, заклинание, золото или +1 к одной характеристике;
- редкий пул с шансом около 10% открывает питомца/фамильяра, спутника или маунта;
- уровень награды: от `max(1, floor(difficulty / 3))` до `difficulty`;
- количество золота равно `уровень награды * 3`;
- награда должна логично соответствовать контексту миссии.
- бот бросает в `reward_roll` редкость, уровень и допустимые типы, а мастер/Codex выбирает конкретный тип и форму по действию героя.

Сложность миссий:

- в ходе должно быть не меньше 3 миссий;
- сложность миссии сравнивается напрямую с успешностью действия героя;
- победа в личной проверке наступает при `успешность >= difficulty`;
- успешность: уровень героя + релевантная характеристика + по одному логично примененному объекту каждого класса;
- допустимый диапазон сложности строится от рейтинга героев: `уровень + средняя характеристика + лучший предмет/заклинание + лучший помощник`;
- стандартное число миссий: `max(3, округление вверх(число персонажей / 3))`.

## Команды админа

```text
загрузить арт фото с подписью /turn_art
загрузить turn_seed.yaml файлом в чат с ботом
загрузить turn.yaml файлом в чат с ботом
/status
/export_turn
загрузить result.json файлом в чат с ботом
/publish_results <turn_id>
/chronicle
```

`/export_turn` остается ручным способом закрыть ход раньше дедлайна.
Если `result.json` положить в `data/imports/pending/`, бот сам импортирует и опубликует его.

## Арт хода

Каждый новый `turn.yaml`, который генерирует Codex, должен содержать общий промпт:

```yaml
turn:
  title: "Ход 3: Пепел над каналами"
  deadline: "2026-05-20 22:00"
  art:
    prompt: |
      Wide cinematic fantasy city illustration of Tanellorn...
    caption: "Ход 3: Пепел над каналами"
```

Чтобы картинка ушла игрокам вместе со стартом хода:

1. Сгенерируй арт по `turn.art.prompt`.
2. Отправь картинку боту как фото или image-документ с подписью `/turn_art`.
3. Затем загрузи `turn.yaml`.

Если картинку не загрузить, бот просто откроет ход и пришлет админу арт-промпт текстом.

## Хроника города

После импорта `result.json` бот сохраняет краткую память о миссиях:

- в SQLite-таблицу `city_chronicle`;
- в `data/chronicle/chronicle.json`;
- в `data/chronicle/chronicle.md`.

В хронику попадают `public_summary`, `world_changes`, статус миссии, название хода и название миссии.
При создании workbench для нового seed бот копирует `chronicle.md` рядом с `lore/`, чтобы Codex учитывал прошлые события при генерации новых миссий.

Админ может посмотреть последние записи командой `/chronicle`.
